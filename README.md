# Генератор перерывов

> *Как пользователь-супервайзер команды,
> я хочу ввести список сотрудников и их смены,
> чтобы система автоматически распределила перерывы по внутренним бизнес-правилам
> и гарантировала корректность расписания без необходимости ручной проверки.*

## Основной сценарий
1. Система получает список сотрудников: ФИО, время начала смены, время окончания смены.
2. Система валидирует входные данные.
3. Система формирует перерывовы, применяя инкапсулированные бизнес-правила.
4. Система возвращает детерминированный результат — корректное распределение перерывов для каждого сотрудника.

## Бизнес-правила генерации
* Набор перерывов определяется продолжительностью смены: при длительности ≥ 9 и < 12 часов назначаются 3 перерыва по 10 минут и 1 обед 30 минут; при длительности ≥ 12 часов дополнительно назначается 1 ужин 20 минут; при длительности < 9 часов перерывы не назначаются.
* Обед для дневных смен должен быть назначен в пределах установленного окна: с 12:30 до 15:00.
* Ужин для дневных смен должен быть назначен в пределах установленного окна: с 17:00 до 19:00.
* Последний перерыв должен быть назначен максимум за 1 час до конца смены.
* Перерывы распределяются так, чтобы сначала заполнялись свободные промежутки, а если сотрудник не помещается, его перерыв назначается параллельно с другим уже запланированным перерывом.
